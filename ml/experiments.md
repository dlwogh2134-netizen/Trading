# ML 실험 기록

본 문서는 LightGBM 기반 주식/코인 신호 모델의 데이터셋, 설정, 성능, 해석을 누적 기록하기 위한 실험 로그입니다.

모델 결과는 매매 실행 명령이 아니라 챗봇의 후보 선별 및 설명 보조 신호로만 사용합니다.

## 2026-06-24 1차 파이프라인 검증

### 목적

* 주식은 가격/거래량 피처만으로는 한계가 뚜렷합니다. 시장지수, 환율, 섹터, 거래대금, 뉴스/수급 데이터 보강이 필요합니다.
* 코인 모델은 종목 수를 늘려도 `구분력(roc_auc)`이 0.63 수준을 유지했습니다. 1차보다 표본이 커졌는데 성능이 유지된 점은 긍정적입니다.
* 코인의 `전체 정답률(accuracy)`은 0.85로 높지만, 상승 라벨이 적은 불균형 데이터일 수 있으므로 단독으로 신뢰하지 않습니다.
* 코인은 다음 실험에서 `4h / 1000` 데이터셋과 `horizon_periods` 조정을 비교할 가치가 있습니다.

## 2026-06-24 3차 하락 위험 모델 추가 및 복합 점수 백테스트

### 목적

* 오르는 종목만 맞히는 `up_label` 단일 모델로는 실제 선별력이 부족한지 확인합니다.
* 떨어질 가능성을 따로 학습하는 `risk_label` 모델을 추가해 `상승 확률 - 하락 위험 확률` 복합 점수가 더 나은 후보 선별을 만드는지 검증합니다.
* 단순 분류 지표와 함께, `signal_score` 상위 3개 후보 기준 단순 백테스트 결과를 같이 기록합니다.

### 왜 하락도 따로 학습하는가

* 상승 확률만 보면 "조금 오를 가능성"과 "크게 빠질 위험"이 섞여 있을 수 있습니다.
* 실제 매매 후보 선별에서는 "오를 가능성"보다 "큰 하락을 피하는 능력"이 더 중요할 때가 많습니다.
* 그래서 이번 실험에서는 아래처럼 모델을 분리했습니다.

```text
상승 모델:
- target = up_label
- 의미 = 예측 기간 뒤 기준 수익률 이상 오를 가능성

하락 위험 모델:
- target = risk_label
- 의미 = 예측 기간 뒤 손실 위험 구간에 들어갈 가능성

복합 점수:
- signal_score = (up_probability - risk_probability) * 100
```

### 추가 모델 성능 결과

#### 주식

| 모델 | 전체 정답률 | 상위후보 적중도 | 구분력 | 해석 |
|---|---:|---:|---:|---|
| 상승 모델 `lgbm_stock_signal_v1` | 0.5352 | 0.4330 | 0.5024 | 랜덤 수준에 가까움 |
| 하락 모델 `lgbm_stock_risk_v1` | 0.6000 | 0.4264 | 0.5357 | 상승 모델보다 약간 낫지만 아직 약함 |

#### 코인

| 모델 | 전체 정답률 | 상위후보 적중도 | 구분력 | 해석 |
|---|---:|---:|---:|---|
| 상승 모델 `lgbm_crypto_signal_v1` | 0.8512 | 0.2007 | 0.6342 | 약한 신호 가능성 유지 |
| 하락 모델 `lgbm_crypto_risk_v1` | 0.8679 | 0.2050 | 0.6596 | 상승 모델보다 약간 더 안정적 |

### 단순 백테스트 조건

```text
- 검증 구간 각 날짜별로 signal_score 상위 3개 후보 선택
- 선택 후보의 미래 수익률 평균 계산
- 같은 날짜 전체 후보 평균 수익률과 비교
- 전략 1: 상승 모델 점수만 사용 (up_only)
- 전략 2: 상승 확률 - 하락 위험 확률 복합 점수 사용 (composite)
```

### 주식 백테스트 결과

| 전략 | 상위 3개 평균 수익률 | 전체 평균 수익률 | 초과수익 | 날짜 승률 | 선택 종목 승률 |
|---|---:|---:|---:|---:|---:|
| `up_only` | 0.0008 | 0.0038 | -0.0029 | 0.5070 | 0.4742 |
| `composite` | 0.0002 | 0.0038 | -0.0036 | 0.4789 | 0.4695 |

### 코인 백테스트 결과

| 전략 | 상위 3개 평균 수익률 | 전체 평균 수익률 | 초과수익 | 날짜 승률 | 선택 종목 승률 |
|---|---:|---:|---:|---:|---:|
| `up_only` | -0.0025 | -0.0018 | -0.0007 | 0.4667 | 0.4547 |
| `composite` | -0.0018 | -0.0018 | -0.0000 | 0.4821 | 0.4513 |

### 해석

* 주식은 `up_only`와 `composite` 모두 전체 평균 수익률보다 낮았습니다. 현재 가격/거래량 중심 피처만으로는 실전 후보 선별력이 부족합니다.
* 주식에서는 하락 위험 모델을 붙였다고 해서 바로 좋아지지 않았습니다. 이는 `risk_label` 추가 자체보다 입력 피처 보강이 더 시급하다는 뜻입니다.
* 코인은 `composite`가 `up_only`보다 나아졌습니다. 초과수익이 `-0.0007`에서 거의 `0` 수준까지 회복됐고 날짜 승률도 소폭 개선됐습니다.
* 다만 코인도 아직 "전체 평균보다 확실히 우수한 전략"이라고 말할 근거는 부족합니다.

### 현재 판단

* 질문에 대한 답은 "떨어지는 구간도 반드시 학습하는 편이 맞다"입니다.
* 다만 하락 학습을 추가했다고 바로 실전 전략이 되는 것은 아닙니다.
* 현재 단계에서는 추가 모델 튜닝보다 아래 데이터 보강이 우선입니다.

2. 코인은 `BTC/ETH 동조`, `거래량 급증`, `변동성 확장`, `김치프리미엄 후보` 피처를 더 강화합니다.
